java-Map详解

发表于 2018-10-21 | 分类于 编程语言 ， java
字数统计: 3.2k | 阅读时长 ≈ 12
Map的相关数据结构，包括HashMap、LinkedHashMap、TreeMap、WeakHashMap，相关的Set也是基于Map实现的

引言

Map在功能上相当于Python中的字典，以键值对<key,value>的形式存储数据，根据实现方式、设计目的的不同有多种Map的实现，包括HashMap、LinkedHashMap、TreeMap、WeakHashMap

HashMap | Set

数据结构

JDK1.8之前，底层实现就是数组+链表，JDK1.8引入红黑树，红黑树和链表都是用于处理哈希冲突的，所以HashMap处理哈希冲突的方法本质上是冲突链表，HashMap是允许key或者value为null的

  1. Node<K,V>[] table，存储节点的数组

  2. size，元素数量，实际存储了多少元素

  3. threshold，阈值，默认是16

  4. loadFactor，负载因子，默认是0.75

    当size>=threshold*loadFactor时扩容

负载因子

或者说 为什么HashMap的扩容不像ArrayList那样是装不下了才扩容

其实就是负载因子为什么默认是0.75，而不是1，其实这是时间和空间的平衡

首先，明确负载因子就是用来决定扩容阈值的，而扩容是解决哈希冲突的有效方法，因此可以认为负载因子比较低，扩容更容易发生，哈希冲突的概率也会有所下降，但是不可否认的是，负载因子越低，数组的空间利用率也就会越低。

综上所述，0.75就是同时兼顾哈希冲突概率和空间利用率的理想值

容量

原始计算出来的哈希值的范围是很大的，冲突基本不会发生，但是内存放不下，因此需要根据数组的长度来做取模运算，得到的余数作为存储位置，对于取模运算，如果除数是2的幂次方，则取模运算等价于(length-1) & hash，采用二进制的按位与操作比取模运算开销小，所以HashMap的大小是2的幂次方，扩容也是两倍来扩容

初始容量

如果入参设置了初始容量，那么需要找到比这个初始容量大或者相等的最小2的幂次方，比如输入17，那就应该是32

源码中对这个计算有所优化，比较重要的是二进制向右移按位或的操作，右移按位或操作的目的在于将二进制的各个位置填上1，这样，得到的数字就是一个2的幂次-1了，比如1111=1+2+4+8=15，最后再把这个1补回来就行了

扰动函数

扰动函数的目的在于优化散列的效果

一般来说，哈希值是int型的，其表示的范围很大，理论上发生哈希冲突的概率是非常小的，但是我们不可能用这么大的数组表示哈希表，所以默认大小是16，那就需要进行取余操作来得到下标，而对2的幂次方取余，可以优化为length-1的按位与操作，那么16-1=15的二进制表示是001111，如果直接按位与，那么只有哈希值的低位发挥作用（高位都是0嘛），按位与操作是不太好改动的，因此引入扰动函数其实就是对将原哈希值的高16位和低16位混合起来，异或后能够保留高位的信息，再按位与能够增大随机性

迁移

迁移数据不需要重新计算哈希值

数据迁移计算元素新位置时，直接看哈希值中新发挥作用的位是不是0，如果为0，则位置不改变，否则右移原始容量的位置

这是因为比如我们正常计算位置，是用hash值和length-1做按位与嘛，而对于2的幂次方来说，length-1，其实就是低位全是1，高位全是0，扩大两倍，只不过是最近的高位从0变成了1，也就是说，对新的位置的计算只需要考虑这个多出来的1，而这个多出来的1反映到哈希值中，就是哈希值多了一位对计算下标有影响的位，这一位如果是0，说明这个多出来的1没有影响，如果是1，说明有影响，且影响刚刚好就是右移原始容量这么多位置

比如原本是16，16-1=15=1111，现在扩容为32，32-1=31=11111，如果哈希值中多出来的这一位对应2^4是1，那么就右移2^4=16，否则不变。

常用方法

put

  1. 首先计算哈希值，同时进行扰动操作

    (key == null) ? 0 : (h= key.hashCode()) ^ (h >>> 16);

  2. 如果数组为null或者长度为0，则需要初始化

  3. 根据哈希值计算下标，如果下标没有存数据，直接存，否则转4

    tab[i = (n - 1) & hash])

  4. 根据这个地方是链表还是红黑树做不同的插入操作

    需要注意链表插入的时候，如果链表长度大于8，需要考虑转换为红黑树

  5. 如果元素数量超过负载时，进行扩容

get

  1. 结合扰动函数计算哈希值

    (key == null) ? 0 : (h= key.hashCode()) ^ (h >>> 16);

  2. table数组是否为空，为空则结束，否则转3

  3. 计算下标

    tab[i = (n - 1) & hash])

  4. 然后就是依据红黑树还是链表进行查找 | 遍历操作了

remove

除了删除key对应项外，还需要返回删除的key对应的value

JDK 1.7底层是数组+链表，所以remove方法的实现如下

  1. 首先根据key计算在数组中的下标

  2. 然后遍历链表，根据key和哈希值找到目标节点

  3. 判断是否是头结点

    • 不是，直接按照链表的方式进行移除
  • 是，将下一个节点放到数组中，相当于更新链表头结点
  1. 返回value

JDK 1.8底层是数组+链表和红黑树，区别主要在于需要根据底层数据结构的不同做不同处理

  1. 首先根据key计算在数组中的下标，得到该位置的节点
  2. 然后判断节点p是不是就是要删除的节点
    1. 是，暂存需要被删除的节点
    2. 不是，看存储结构是红黑树还是链表分别遍历，寻找并暂存需要被删除的节点
  3. 删除找到的节点
  4. 返回被删除的节点

遍历

性能由高到低

  1. 迭代器 直接获取EntrySet，通过hasNext进行遍历

  2. 增强for循环获取EntrySet

    上述1 2底层生成的代码实际上一致

  3. Stream API

  4. 迭代器 获取KeySet

  5. 增强for循环获取KeySet

    上述4 5生成的代码一致

  6. lambda:forEach遍历key和value

为什么Map.Entry比KeySet快呢？

因为获取KeySet返回key遍历了一遍，通过map.get(key)获取所有的value又要去查一遍

而Map.entry是一次查询就保存了下来，不需要第二次遍历

安全性

迭代器方式可以边遍历，边删除

增强for循环、lambda不行，lambda可以先使用removeIf进行删除，stream可以用filter进行过滤，再遍历

HashSet

HashSet实际上就是对HashMap做了一层包装，也就是说HashSet内部持有一个HashMap，对HashSet的方法调用基本都会转为对HashMap的方法调用，但是需要注意的是，HashSet的去重功能依赖于HashMap的key，所以HashSet的value在HashMap中是key，value则是同一个final static的Object对象

LinkedHashMap

额外设置LinkedList进行有序性功能增强的HashMap，自然也是没有进行多线程安全控制的，具体来讲就是通过双向链表将所有entry进行连接，保证元素的迭代顺序和插入的顺序一致，同时遍历时间只与entry的数目相关，和table的大小无关

Entry实际上是Node的子类，并扩展了before和after两个指针用于维护双向链表

put

  1. 从哈希表角度看，冲突链表尾插法
  2. 从双向链表角度看，插入新元素，是插入到双向链表的尾部

FIFO缓存

LinkedHashMap提供一个叫removeEldestEntry的方法用来设置是否要移除最老的项，也就是最早插入的项，只要继承LinkedHashMap并将其重写比如元素数量超过多少就返回true，这样LinkedHashMap会自动删除head指向的元素，实际上就是最先插入的元素，这样就能实现FIFO的缓存策略

LRU缓存

在FIFO实现的基础上，只要将accessOrder设置为true，那么访问一个节点后，afterNodeAccess方法就会自动将该节点移到链表的最后，相当于就是最新插入的项，这样就实现了LRU的删除最久未使用的策略

LinkedHashSet

LinkedHashSet也是对LinkedHashMap进行了一层包装，LinkedHashSet持有一个LinkedHashMap的成员变量，对LinkedHashSet的方法调用实际上就是对LinkedHashMap的方法的调用

WeakHashMap

Weak的意思就是HashMap中对key的引用是一种弱引用，弱引用的特性在于不管内存空间是否足够，只要垃圾回收器扫描到仅有弱引用指向的对象，就会立马将其回收，所以WeakHashMap可以用来做缓存，由垃圾回收器管理缓存对象的有效时间，缓存命中可以提高访问效率，miss也不会报错，再去查就行了

不存在WeakHashMap，但是可以通过

封装一个WeakHashSet

TreeMap

TreeMap底层用红黑树实现，使得大部分方法的时间开销都是O(logn)，并且红黑树本身是一种二叉搜索树，自然能够维护顺序，该顺序是基于key的自然顺序或者传入Comparator的自定义顺序

红黑树

红黑树是一种近似平衡的二叉查找树，它能够确保任何一个节点的左右子树的高度差不会超过二者中较低那个的一倍

特点

  1. 节点要么是红，要么是黑
  2. 根节点是黑，所有叶子节点（Null）是黑的
  3. 红色节点不能连续出现
  4. 对于每个节点，从该节点到叶子节点，黑色节点的个数一致，所以新插入的节点必须是红色

调整

左旋、右旋、改颜色，左旋就是将x的右子树逆时针旋转，将右孩子节点变为x的父节点，x变为左孩子，然后还涉及到一些引用的修改，右旋就是反过来

节点后继

节点后继就是树中大于t的最小元素

  1. 如果有右子树，就是右子树的最小元素
  2. 没有右子树，就是祖先节点中第一个向左走的节点

TreeSet

TreeSet也是对TreeMap进行包装
